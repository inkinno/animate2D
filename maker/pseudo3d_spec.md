# ⚙️ Animate2D Pseudo-3D (의사 3D) 5대 축 보간 규격서

이 문서는 Animate2D Maker Suite에서 2D 평면 에셋(스프라이트)을 다차원 각도에서 바라본 것처럼 입체 변형시키는 **Pseudo-3D (의사 3D) 엔진 모듈(`pseudo3d.js`)의 가중치 보간 알고리즘 및 5대 축 규격**을 기술한 세부 사양서입니다. 

추후 5개 각도 축을 다른 세분화된 각도로 수정하거나 확장(예: 7대 축, 15도 간격 세분화)하고자 할 때, 이 가이드 문서를 편집하고 코드 변수를 조절하면 즉각 반영할 수 있도록 설계 세부사항을 완전 격리 정리하였습니다. (옵션 A 최종 채택)

---

## 📐 1. 5개 기본 각도 축 규격 명세

의사 3D는 2D 스프라이트 씬에 입체적인 뎁스(Depth)감을 주기 위해, 미리 정의된 **5가지 각도 스펙 [-90°, -45°, 0°, 45°, 90°]**을 기준 축으로 설정하고 각 각도 지점에서의 스프라이트 변형 가중치를 수학적으로 보간(Interpolation)합니다.

| 각도 축 구분 | 회전 각도 (deg) | 수학적 매핑 가중치 ($W$) | 시각적 변형 거동 | 주요 쓰임새 |
| :--- | :--- | :--- | :--- | :--- |
| **극좌측 프로필 (Far Left)** | $-90^\circ$ | $-1.0$ | 완전히 뒤돌아보거나 좌측면 최대 수축 | 완전 좌향 모션, 극단적 회전 시작점 |
| **반좌측 사각 (Left Diagonal)**| $-45^\circ$ | $-0.5$ | 좌측 사선 $45^\circ$ 반전 비틀기 | 쿼터 뷰 좌향 모션, 입체적 보폭 묘사 |
| **정면 정렬 (Frontal Origin)** | $0^\circ$ | $0.0$ | 정면 기준 비틀림 왜곡 0% (기본값) | 정면 공격, 대기(Idle) 기본 중심 상태 |
| **반우측 사각 (Right Diagonal)**| $45^\circ$ | $0.5$ | 우측 사선 $45^\circ$ 입체 왜곡 | 쿼터 뷰 우향 모션, 입체적 보폭 묘사 |
| **극우측 프로필 (Far Right)**| $90^\circ$ | $1.0$ | 완전히 우측을 향하거나 우측면 최대 수축 | 완전 우향 모션, 극단적 회전 끝점 |

## 🧮 2. 실시간 가중치 보간 (Interpolation) 알고리즘

사용자가 저작도구에서 정교한 세부 조절 타이핑을 통해 임의의 특정 각도 $\theta$ (예: $-32^\circ$ 또는 $55^\circ$)를 수치상으로 입력하면, 엔진은 이를 기준 축 사이의 선형 보간(Linear Interpolation, LERP) 비율로 치환하여 어핀 행렬에 곱 연산합니다.

### 1) 구간 판독 공식
입력 각도 $\theta$가 주어지면, 아래 4개 보간 구간 중 어디에 속하는지 판독합니다.
1. **구간 1**: $-90^\circ \le \theta < -45^\circ$
2. **구간 2**: $-45^\circ \le \theta < 0^\circ$
3. **구간 3**: $0^\circ \le \theta < 45^\circ$
4. **구간 4**: $45^\circ \le \theta \le 90^\circ$

### 2) 보간 비율 ($t$) 및 가중치 ($W$) 산출 공식
각 구간에서의 정규화된 보간 계수 $t \in [0, 1]$와 최종 변형 강도 가중치 $W$는 다음과 같이 도출됩니다.

$$t = \frac{\theta - \theta_{start}}{\theta_{end} - \theta_{start}}$$

$$W = W_{start} \cdot (1 - t) + W_{end} \cdot t$$

예를 들어, 사용자가 세부조절 수치로 **$\theta = -30^\circ$**를 타이핑하여 입력한 경우:
* 속한 구간: **구간 2** ($-45^\circ$에서 $0^\circ$ 사이)
* $\theta_{start} = -45^\circ$, $\theta_{end} = 0^\circ$
* $W_{start} = -0.5$, $W_{end} = 0.0$
* 보간 비율 $t = \frac{-30 - (-45)}{0 - (-45)} = \frac{15}{45} = 0.333$
* 최종 가중치 $W = (-0.5) \cdot (1 - 0.333) + (0.0) \cdot 0.333 = -0.333$

이 $W$값은 `pseudo3d.js` 내에서 슬롯의 3x3 변환 행렬 중 $m_{00}$ (가로 스케일 변형 인자) 및 $m_{01}$ (기울기 변형 인자)에 연산 인수로 작용하여 부드러운 입체 회전을 구현합니다.

## 🛠️ 3. 추후 각도 규격 커스텀 수정/확장 가이드

추후 뼈대 모델링의 해상도를 높이기 위해 각도를 더 잘게 쪼개거나 다른 고정 규격으로 개편하고자 할 경우, 아래 절차에 따라 파일을 수정하십시오.

### 1) 기준 데이터 구조 정의 (`maker/js/pseudo3d.js` 내 수정)
기본 설정 구조가 DRY 원칙에 따라 분리되어 있으므로, `Pseudo3DEngine` 클래스의 각도 테이블 스펙 변수 배열만 변경하면 전역 보간 루프가 자동 갱신됩니다.

```javascript
// 기존 5대 축 구조
this.angleAxes = [-90, -45, 0, 45, 90];
this.weightAxes = [-1.0, -0.5, 0.0, 0.5, 1.0];

// 예시: 7대 축 세밀 고도화 시 수정본
// 15도/30도 간격의 극세화 보간 스펙
this.angleAxes = [-90, -60, -30, 0, 30, 60, 90];
this.weightAxes = [-1.0, -0.66, -0.33, 0.0, 0.33, 0.66, 1.0];
```

### 2) 3x3 어핀 행렬과의 기하적 사상 연동
가중치 $W$에 따른 최종 x축 전단 변형(Shearing Offset)과 입체 스케일 인장비는 다음 어핀 변환으로 매핑되어 자식 관절에 상속(FK)됩니다.

```javascript
// W는 보간된 가중치 (-1.0 ~ 1.0)
const skewX = Math.sin(W * Math.PI / 4) * 0.25; // x축 왜곡도
const scaleX = Math.cos(W * Math.PI / 4);       // 원근 압축률

// 3x3 행렬 갱신 (matrix.js의 정적 풀 인플레이스 연산 결합)
matrix[0] = scaleX; // m00
matrix[1] = skewX;  // m01
```

이 규칙은 `basic center`이자 최핵심 파트 오리진인 `👑 ROOT ORIGIN`에서부터 계산되어 자식 본 노드들로 유기적으로 흘러내려가며 무손실 가속 3D 효과를 구현합니다.
