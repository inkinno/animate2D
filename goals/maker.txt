[Document 4] 메이커 기획서 : 웹 기반 타임라인 저작 도구 및 다방향성 각도 보간 명세서 (Maker Editor Specification)
1. 다방향성 의사 3D 보간 및 피봇 정렬 가중치 시스템 (Pseudo-3D Engine)
메이커는 완전한 3D 공간을 연산하지 않고 평면 이미지(WebP)를 다루되, 공간적 입체감을 의사(Pseudo) 표현하기 위해 '다방향성 각도 보간 가중치 시스템(Pseudo-3D Directional Weighting)'을 엔진 스키마와 연동한다.

초기 MVP 사양에서는 전면(Front), 측면(Profile), 반측면(Three-Quarter)을 포함한 핵심 5개 각도(Directional Angle) 상태 레이어를 기준 축으로 설정한다.

각 장비, 옷, 갑옷 및 하위 무기 부속 조각들은 타임라인 상에서 독립된 각도 세그먼트를 보유한다. 작가가 타임라인에서 정해진 기준 각도 사이의 '임의의 중간 각도(예: 32도)'를 입력하면, 메이커의 보간 엔진은 해당 각도에 인접한 앞/뒤 평면 에셋 소스의 불투명도(Opacity)와 변형 Matrix 스케일을 수학적 가중치로 자동 연산하여 픽셀이 입체적으로 회전하는 듯한 공간 착시 연출을 유도한다.

2. 수치 기반 단축키 포커싱 조작계 및 마우스 드래그 배제 (Numeric Focus UI)
메이커의 중앙 작업 뷰포트에서는 화면 왜곡과 연산 오류를 방지하기 위해 마우스를 드래그하여 본의 위치나 크기를 임의 변경하는 마우스 변형 인터랙션(Mouse-driven Transform Gizmo)을 원천 차단하고 폐기한다.

작가가 작업 화면 내 특정 파츠를 마우스로 클릭하면, 해당 파츠 경계면에 소프트 하이라이트(Glow CSS Outline)만 표출되며, 메인 마우스 포인터의 화면 이동 권한은 잠금(Lock) 처리된다.

파츠 선택 즉시 우측 프로퍼티 패널(Property Panel)의 회전 각도(Rotation) 입력 박스로 포커스(Focus)가 자동 강제 이식되며, 작가는 키보드 숫자를 통해 정밀한 수치 수정을 집행한다.

패널 내부에는 수치 입력의 고속화를 위해 단축키 파이프라인을 심는다. 각도 입력 상태에서 키보드 'Tab' 키를 누르면 다음 연산 인수인 가로/세로 스케일(Scale) 입력 필드로 커서가 즉시 컨텍스트 전이(Context Transition)되어 마우스 조작 없이 양손 키보딩만으로 타임라인 변형값 수정을 완료할 수 있게 설계한다.

3. 스케일 연동형 자동 레이어 제어 및 잠금 매커니즘 (Auto-Layering Guard)
어깨에 붙어 있는 팔뼈와 같이 계층 구조로 연결된 파츠들 사이의 시각적 결합 안정성을 위해 '스케일 연동형 자동 레이어 계층 시스템'을 탑재한다.

오리진(Origin) 부모 본에서 자식 본으로 넘어가는 결합부 회전 피봇 지점에서, 자식 본의 런타임 스케일 인수가 부모 본의 고유 스케일 인수보다 '같거나 작게' 수치 조절될 경우, 엔진은 결합부 파편화를 방지하기 위해 부모 본 영역에 자동으로 더 높은 레이어 우선순위(Higher Z-Index Priority)를 연산 배정한다.

수치 입력에 의한 원치 않는 데이터 오버라이트(Overwrite)를 방지하기 위해, 각 파츠 객체의 개별 속성(위치, 크기, 레이어 순서) 우측면에 각각 독립된 '수동 락킹 버튼(Lock Toggle Component)'을 배치하여 레이아웃 값이 고정된 상태를 강제할 수 있도록 지원한다.

4. 4분할 레이아웃 GUI 아키텍처 (Workspace Layout Template)
메이커의 웹브라우저 인터페이스 화면은 작업 가시성 최적화를 위해 철저하게 다음의 4분할 영역으로 고정 배치 구조를 가진다.

1 영역 - 좌측 관리형 탐색기 사이드바 (Left Navigator Panel): 프로젝트 관리 트리(Project Tree), 하부 세부 씬 리스트 관리자(Scene List Manager), 그리고 씬과 씬 사이를 매끄럽게 연결하는 전환 탭 관리자(Transition Interface Tab)가 수직 적층(Vertical Stacking) 구조로 좌측 라인에 고정 배치된다.

2 영역 - 중앙 실시간 뷰포트 (Center WebGL Viewport): 가용 면적 계산 공식에 따라 도출된 고정 종횡비 캔버스가 완벽히 상단 정렬(Top Alignment)된 상태로 실시간 렌더링 피드백을 송출하는 공간이다. 마우스 변형은 작동하지 않으며 오직 시각적 결과 체크 기능만 수행한다.

3 영역 - 우측 속성 데이터 편집 패널 (Right Property Inspector): 선택된 본과 슬롯의 절대/상대 좌표 수치, 각도 값, 스케일 팩터, 아틀라스 소스 매핑 데이터가 수치 폼(Form) 형태로 표출되며, Tab 키 포커싱 제어 장치가 실행되는 메인 입력 장치이다.

4 영역 - 하단 타임라인 시퀀서 및 계층 종속성 가이드 (Bottom Timeline Sequencer): 전체 재생 바(Scrubber)와 밀리초(ms) 단위의 트랙 레이어가 수평 배치된다. 타임라인 내 각 파츠 노드의 이름 옆에는 해당 파츠가 현재 역학 구조상 쫓아가고 있는 '종속된 부모 파츠의 ID 이름'을 인디케이터 텍스트로 상시 표출하여 트랙 간의 계층적 종속성(Hierarchy dependency)을 한눈에 식별할 수 있도록 설계한다.

5. 저작 편의성 극대화를 위한 워크플로우 제안 (Workflow Enhancements)
트랜스폼 데이터 클립보드 스택 (Transform Clipboard Stack): 작가가 씬 1에서 공들여 세팅한 특정 관절 본들의 수치 세트(위치, 크기, 각도 전체)를 'Ctrl + C'로 가상 클립보드 버퍼 메모리에 복사한 뒤, 씬 2의 다른 시간축 프레임이나 전혀 다른 캐릭터의 대칭 관절 본에 'Ctrl + V'로 매트릭스 수치 값을 그대로 이식 인젝션할 수 있는 데이터 복사/붙여넣기 편의 기능을 구현한다.

어니언 스킨 가이드 필터 (Onion Skinning Filter): 타임라인에서 키프레임 수치 입력을 수행할 때, 직전 키프레임 지점(T-1)과 직후 키프레임 지점(T+1)에 배치된 스프라이트 파츠의 위치 실루엣을 작업 화면에 반투명한 잔상(알파값 20% 레이어)으로 가이드 투영해 준다. 이를 통해 메쉬가 없는 단순 수치 트랜스폼 환경에서도 관절이 끊기거나 비례가 무너지는 실수를 시각적으로 방지하여 수치 입력 작업의 정밀도를 비약적으로 상승시킨다.

실시간 미분 이징 프리뷰어 (Easing Graph Monitor): 우측 속성 패널에서 키프레임 보간 수치(Bezier 인수)를 수정할 때, 하단 타임라인 스크러버를 수동으로 밀어보지 않아도 해당 관절 파츠가 가속/감속되는 가상 운동 에너지 곡선 그래프를 팝업 썸네일로 미세 가이드해 준다. 작가는 수치 입력과 동시에 가속도의 곡선 궤적을 직관적으로 보정할 수 있다.
